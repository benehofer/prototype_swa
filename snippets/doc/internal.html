Internal API section

This set of API endpoints is for the SNOW side. The API offers functions to create and manage incidents and subscriptions. The incident information will be forwarded to the customer. Subscriptions allow to to subscribe to incident events.

Incident resource type

API to create, retrieve and delete incidents. An incident record can contain worknotes and attachments.

URL variables
Variable name Required Description
subscriptionid False The internal ID of the subscription
Response body

  {
    "expirationdate": "07/10/2023 03:01:38",
    "subscriptionid": ,

Response body fields
Variable name Description
expirationdate The expiration date of the subscription; if no value is specified, the subscription will be active infinitely
subscriptionid The internal ID of the subscription
changetype The monitored change type of the subscription; this value is either 'Created' or 'Modified'
notificationheaders The headers to be used when sending post messages to the endpoint; the value must be a json string representation of the headers
clientstate The client state attribute of the subscription; this can be used to verify post messages to the endpoint. Each post message will include the value specified here
notificationurl The url to be used for the webhook messages optionally including query strings
resource The resource to be monitored for changes; currently only changes in resource 'incident' are available.
