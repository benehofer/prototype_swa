API Manage section

This section of the API is for admin purposes. You can register and manage customers.

Customer

The customer resource type represents a customer in the API. Customers interact with the resources in the public section of the API. The customer resource is the representation of the corresponding resource in the backend system.
A specially-configured customer resource is created to access the internal area.

Retrieves an exiting customer record or all active customer records.

Customer secret

The customer secret resource type creates the authentication token for a customer resource. The token contains detailed information about the customer resource. A new secret must be generated each time a change is made to the customer data. The secret must be included in the http header of the call.

get

Retrieves the secret (token) of a customer record. The token includes the allowed source ip ranges and the allowed destination urls for the customer.

get /api/manage/customer/{customerid}/secret
URL variables
Variable name Required Description
customerid True
Response body

  {
    "customersecret": dHGHTkjkhJHjJHjIhhjHjHJgRtdFdGGKJHLKH...
  }
Response body fields
Variable name Description
customersecret The secret/token for the customer record.

Allowed source ip ranges

The allowedIP resource type defines the allowed source IP ranges for a client. Each request to the API is checked to see if it comes from one of the allowed ranges. Multiple ranges can be registered for a customer resource.

get

Retrieves an allowed ip or all allowed ip ranges from a customer record.

get /api/manage/customer/{customerid}/allowedip/{allowedipid}

Allowed destination (API) urls

The allowedURL resource type defines allowed resources of the API for a customer. For this purpose, urls of the API endpoints are captured; wildcards can be used. For each call, it is checked whether the URL is permitted for the respective customer.
